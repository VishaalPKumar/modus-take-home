Valuation Engines

Comparable Company Analysis

How it works

  • Find public comps by sector
  • Calculate EV/Revenue multiples for each
  • Apply median multiple to target company revenue

Implementation

  • statistics.median() for central estimate
  • quantiles() for Q1/Q3 range
  • Minimum 2 comparables required (raises ValueError)

 Mock Data

15 real public companies across 4 sectors

Datadog, Snowflake, CrowdStrike, Palantir, UiPath…

 Output

Point estimate + interquartile range

Step-by-step derivation with citations

Discounted Cash Flow (DCF)

How it works

  • Project 5-year free cash flows
  • Discount at WACC to present value
  • Add terminal value via Gordon Growth Model

Key Formulas

  • Year N FCF = Revenue × (1+g)N × margin
  • Terminal = FCFfinal × (1+gt) / (WACC − gt)

 Validation

Cross-field: discount_rate must exceed terminal_growth_rate

Range: ±20% sensitivity bounds

 Why DCF Matters for Audit

Most assumption-heavy methodology — most important to make every input transparent and traceable

Last Round (Market-Adjusted)

How it works

  • Start from last funding round valuation
  • Adjust by public index performance since that date
  • Simplest method — provides useful anchor point

Implementation

  • Uses Nasdaq index data (mock: Jan 2023 – Mar 2026)
  • Index return = (current − round_date) / round_date
  • Range: ±15% for illiquidity / company-specific risk

 Date Validation

Requires index data within 180 days of round date

Raises error if no suitable data point found

 Role in Triangulation

Provides market-based anchor to cross-check model-driven estimates from Comps and DCF

Sensitivity & Triangulation

Sensitivity Analysis

  • DCF: 2D grid (discount rate × growth rate) with color-coded heatmap
  • Comps / Last Round: 1D parameter sweep
  • Shows how the estimate changes with different assumptions

 Why It Matters

Auditors want to know: “How sensitive is this number to my assumptions?”

Triangulation (Multi-Method)

  • Run all 3 methods simultaneously
  • Comparison table: method, estimate, range
  • Visual range bars with convergence zone overlay

 Convergence

If ranges from multiple methods overlap → higher confidence in the estimate

Divergence flags areas needing closer review

Key Design Decisions

 Strategy Pattern

Pluggable valuation engines — easy to add new methodologies without touching existing code

 Pydantic as Source of Truth

One model definition drives validation, serialization, and OpenAPI documentation

 DataProvider ABC

Mock data today, real APIs tomorrow — swap without touching business logic

 Audit-First Design

Every calculation step recorded, not just final numbers — built for traceability from the start

 Multi-Method Triangulation

Convergence analysis across methods increases confidence — divergence flags areas for closer review
